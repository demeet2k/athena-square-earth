# TANDEM AGENTIC LOOP PROTOCOL

## Purpose

This protocol captures the user's clarification that the prompt is meant to be broadcast
to many agents in tandem so they cooperate on one shared build rather than behaving like
isolated monologues.

## Core Rule

The same high-level prompt may be given to many agents, but those agents are not
independent authors. They are cooperative workers inside one shared neural-network build.

## Tandem Law

`shared objective + distinct shard + common packet schema + deterministic contraction`

The agents cooperate correctly only when:

1. they share one task identity,
2. each receives a distinct shard or role,
3. each emits packets that others can consume,
4. all packets contract back into a common synthesis target.

## Broadcast Structure

Recommended tandem tiers:

1. `kernel agent`
   - defines task, corpus slice, stop rule, and contraction target
2. `4 pillar agents`
   - Earth, Water, Fire, Air
3. `16 archetype agents`
   - mixed second-order roles
4. `64 worker agents`
   - local shard readers, pattern extractors, route builders, witness checkers

## Cooperation Rules

1. no worker claims whole-corpus authority,
2. no worker skips witness fields,
3. all workers inherit the same output schema,
4. conflicts are preserved until contraction decides them,
5. the kernel owns final merge and next-seed emission.

## Output Contract

Every tandem agent should emit:

- address,
- source shard,
- retained claims,
- unresolved ambiguities,
- contraction target,
- replay hint.

## Failure Modes

Common tandem failures:

1. duplicated effort without shard separation,
2. incompatible output formats,
3. premature whole-system conclusions,
4. missing contraction ownership,
5. lost ambiguities during merge.

This protocol exists to prevent those failures.
