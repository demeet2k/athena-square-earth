# CRYSTALLINE PROCESSING UNIT (CPU)

## Purpose

This directory defines a document-native compute framework: a Crystalline Processing Unit
(CPU) that turns static manuscripts into active routing, planning, synthesis, and governance
surfaces. The framework is designed for tasks that begin in text, expand into parallel
agents, and contract back into one witnessed artifact.

The CPU is grounded in four corpus patterns already present in this workspace:

1. Voynich demonstrates staged curriculum, modular books, and executable crystal summaries.
2. Torah-adjacent Kabbalah and Merkavah material demonstrate address grammar, guarded ascent,
   letter-force ontologies, and gate-based progression.
3. Nostradamus demonstrates compressed operator packets, timing gates, and multi-layer
   decoding under ambiguity.
4. The existing ECOSYSTEM layer already defines routing, witness classes, and 4^4 agent logic.

## Core Claim

Text behaves like compute when five conditions are satisfied at the same time:

1. every unit has a stable address,
2. every transition is governed by an explicit operator,
3. every expansion can contract back into a smaller seed,
4. every mutation carries witness and replay obligations,
5. every agent wave is governed by promotion and conflict rules.

## Operational Depth

The default CPU compile depth is:

- `1 -> 4 -> 16 -> 64`

This yields:

- `1` coordinating kernel,
- `4` elemental pillars,
- `16` archetypal agents,
- `64` active worker agents.

An optional fourth split may be added later:

- `1 -> 4 -> 16 -> 64 -> 256`

The CPU therefore treats `64` as the standard operating crystal and `256` as the
high-resolution extension.
