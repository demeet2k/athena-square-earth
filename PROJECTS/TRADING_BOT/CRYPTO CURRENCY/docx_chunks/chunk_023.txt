□ B.S — Square: EvidenceRow, EvidenceID, and Append-Only Evidence Tables
B.S1 — Objects
B.S1.a — EvidenceRow (normative schema)
An EvidenceRow is the minimal canonical record that binds a claim to replayable evidence under pinned interpretation.
[
\mathsf{EvidenceRow} :=
\langle
\mathsf{EID},;
\mathsf{ClaimAddr},;
\mathsf{WitnessType},;
\mathsf{Scope},;
\mathsf{TruthType},;
\mathsf{PinsDigest},;
\mathsf{ReplayPtr},;
\mathsf{WitnessPtr},;
\mathsf{InputDigests},;
\mathsf{OutputDigests},;
\mathsf{Metrics},;
\mathsf{IssuedPulse},;
\mathsf{ExpiryPulse?},;
\mathsf{Supersedes?},;
\mathsf{SchemaVer}
\rangle
]
Required fields:
EID: EvidenceID (defined in B.S2.a).
ClaimAddr: canonical GlobalAddr of the subject claim (node/edge/route/certificate).
WitnessType: pinned enum describing what the row proves, at minimum:
ReplayWitness (replay capsule proves claim),
EquivWitness (EQUIV proof object),
DualWitness (DUAL proof object),
MigrationWitness (MIGRATE proof object),
ResidualLedger (NEAR bounds),
EvidencePlan (AMBIG plan),
ConflictPacket (quarantine/fence evidence),
AuditWitness (audit proof object).
Scope: pinned enum {LOCAL, INTER} plus optional scope bindings:
LOCAL: manuscript_id required.
INTER: ordered pair (src_manuscript_id, dst_manuscript_id) required.
TruthType: pinned enum {OK, NEAR, AMBIG, FAIL, CONFLICT}.
PinsDigest: digest of the complete pin-closure required to interpret this row (AppA/AppD/AppI).
ReplayPtr: pointer to replay capsule (or deterministic replay recipe) sufficient to reproduce the verdict. Required for OK and NEAR; optional for AMBIG when an EvidencePlan is the only object; required for FAIL/CONFLICT when reproducible.
WitnessPtr: pointer to the primary witness artifact for the WitnessType.
InputDigests: canonical ordered list of digests of inputs required by replay (containers, profiles, policies, codebooks).
OutputDigests: canonical ordered list of digests of outputs claimed by the witness.
Metrics: optional pinned metric outputs (residual bounds, commutation residuals) with metric digests and units. Mandatory for NEAR.
IssuedPulse: PulseID at which this row is issued (Ch20).
ExpiryPulse?: PulseID at which this row ceases to qualify for OK routing under the stability contract (mandatory for NEAR, optional otherwise).
Supersedes?: optional EvidenceID of a prior row that this row supersedes (append-only supersession).
SchemaVer: evidence schema version (pinned, immutable).
B.S1.b — EvidenceTable
An EvidenceTable is an append-only multiset of EvidenceRows with canonical ordering and index structures.
[
\mathsf{EvidenceTable} :=
\langle
\mathsf{TableID},;
\mathsf{PolicyDigest},;
\mathsf{CanonBytesDigest},;
\mathsf{RowsDigest},;
\mathsf{RowIndexDigest},;
\mathsf{PulseDigestRef},;
\mathsf{SchemaVer}
\rangle
]
RowsDigest commits to the canonical list of rows.
RowIndexDigest commits to canonical indices (by claim addr, by EID, by truth type, by witness type, by pulse).
PulseDigestRef binds the table state to the pulse digest chain (Ch20).
B.S1.c — Index invariants
Indices are derived; the table commits to a canonical index digest. Index derivation must be deterministic and replayable:
Index key order: canonical lexicographic order over serialized keys.
Collision resolution: deterministic and logged.

A StabilityContract is a pinned rule set specifying when an EvidenceRow may be treated as stable for routing under given corridor requirements.
[
\mathsf{StabilityContract} :=
\langle
\mathsf{ContractID},;
\mathsf{Pins},;

B.S2 — Operators
B.S2.a — EvidenceID derivation (pinned)
EvidenceID MUST be content-derived and collision-resistant under pinned canonicalization.
Define:
(R := \mathsf{ser}(\mathsf{EvidenceRowZeroed})) where EID and any signature fields are zeroed and Supersedes? is preserved.
(k :=) a pinned domain-separation tag "AppB/EvidenceRow".
Then:
[
\boxed{
\mathsf{EID} = \mathsf{H}\big(k ,|, \mathsf{H}(R) ,|, \mathsf{PinsDigest}\big)
}
]
where (\mathsf{PinsDigest}) is the digest of the pin-closure referenced by the row.
Collision handling.
If an implementation detects (\mathsf{EID}) collision with a distinct row (distinct canonical bytes), it MUST:
Treat as integrity failure (typed CONFLICT),
Emit a ConflictPacket witness row (WitnessType=ConflictPacket) describing:
both row digests,
collision digest,
reproduction procedure,
Quarantine both rows for OK routing until resolved by schema/policy migration.
No silent disambiguation suffixes are permitted; any disambiguation must be explicit via migration and pinned rule changes.
B.S2.b — Row insertion and append-only discipline
EvidenceTable update is a DeltaPack operation. Insertion is append-only:
Rows are appended, never rewritten.
Supersession is represented only by a new row referencing Supersedes?.
Let (T_p) be the evidence table at pulse (p). Applying a DeltaPack at pulse (p+1) yields:
[
T_{p+1} = T_p \cup \Delta T_{p+1}
]
where (\Delta T_{p+1}) is the set of new rows.
B.S2.c — Canonical row ordering in table commits
EvidenceTable commits to a canonical ordering of rows by a pinned comparator:
Primary key:
IssuedPulse ascending
ClaimAddr canonical order
TruthType order (OK > NEAR > AMBIG > FAIL > CONFLICT) for deterministic grouping
WitnessType canonical enum order
EID lexicographic
This ordering is used to compute RowsDigest. Any change to ordering rules is a meaning-bearing policy change and requires explicit migration.
B.S2.d — Merge operator for evidence tables
Evidence tables merge only through DeltaPack application and deterministic merge rules. For merging two sources (T^A, T^B) into (T^{A\cup B}), define:
merge input: two row streams and their pulse ranges
merge output: a unified row stream with deterministic conflict detection
Merge algorithm:
Validate chain integrity and policy compatibility (AppD/Ch20).
Collect all rows, deduplicate by EID with exact equality of canonical row bytes.
If two distinct rows share an EID, emit a merge conflict and quarantine (see B.S2.a collision handling).
Recompute indices deterministically.
Emit MergeDeterminismCert.

B.S3 — Invariants
B.S3.a — EvidenceRow well-formedness
An EvidenceRow is well-formed iff:
ClaimAddr parses under AppA grammar and resolves under pinned registry.
PinsDigest resolves to a complete pin-closure required by the witness type.
ReplayPtr presence matches truth type requirements:
OK: required
NEAR: required
AMBIG: optional, but required if any deterministic selection is asserted
FAIL/CONFLICT: required if reproducible, optional only if policy explicitly permits non-reproducible fails (not permitted in publish corridors)
Metrics is present and complete if TruthType=NEAR.
IssuedPulse is valid under PulseID policy.
EID equals the pinned derivation function output.
B.S3.b — Stability contract soundness
If a StabilityContract marks an EvidenceRow as usable for OK routing, then:
The row must be TruthType=OK
Replay must close under the pinned replay harness, and all referenced digests must validate
The row must not be expired under ExpiryPolicy
The row must not be quarantined by any applicable quarantine/ revocation policy
B.S3.c — Expiry and renewal monotonicity
Expiry is monotone in time: once expired, a row cannot become unexpired. Renewal occurs only via a new EvidenceRow that supersedes prior evidence and references a later pulse and later replay.
B.S3.d — Migration compatibility invariants
If schema/policy pins change, evidence stability must be re-evaluated under MigrationPolicy:
Rows referencing obsolete schema pins are not OK-routable unless explicitly migrated.
Migration is represented by new rows with WitnessType=MigrationWitness and TruthType per corridor evaluation.

B.S4 — Protocols
B.S4.a — Evidence issuance protocol
To issue evidence for a claim:
Determine ClaimAddr and Scope.
Select WitnessType and produce WitnessPtr and ReplayPtr.
Compute PinsDigest for the complete closure required to interpret the witness and to replay verification.
Compute input and output digests.
Determine TruthType:
OK: replay closes, digests match, invariants hold exactly
NEAR: bounded residual, metrics provided, expiry defined, obligations recorded
AMBIG: evidence plan or bounded candidate set, no unpinned tie-breaks
FAIL/CONFLICT: deterministic failure witness or conflict packet
Compute EID.
Append as DeltaPack row insertion at IssuedPulse.
B.S4.b — Evidence lookup protocol
Given a claim address and a corridor requirement, lookup proceeds:
Retrieve all rows matching ClaimAddr.
Filter by Scope compatibility.
Filter by valid pins under the current policy epoch (AppD/Ch20).
Apply StabilityContract to determine which rows are admissible under the corridor.
Choose the best admissible row by pinned preference order:
prefer OK over NEAR over AMBIG over FAIL over CONFLICT
prefer latest IssuedPulse subject to non-expiry
tie-break by EID
Emit selected evidence row plus any required disclosures (NEAR metrics) or evidence plan (AMBIG).
B.S4.c — Table merge protocol under DeltaPacks
Merge is performed by applying imported DeltaPacks and reconstructing the EvidenceTable deterministically. The merge procedure must:
validate import integrity (digests, signatures),
validate pulse chain integrity,
validate row schemas and EIDs,
detect collisions and conflicts deterministically,
re-emit indices and RowsDigest,
emit MergeDeterminismCert.
B.S4.d — NEAR renewal protocol
If a row is NEAR and ExpiryPulse is reached (or imminent under policy), renewal is performed by issuing a new row with Supersedes? set to the prior EID and with one of:
OK upgrade via new replay closure, or
NEAR renewal via tightened bounds or extended evidence, with a new expiry and replay pointer.
Renewal must be deterministic and append-only.

□ C.S — Square: 4^4 Tile Semantics, Address Grammar, Local Links, Local Proof Objects
C.S1 — Objects
C.S1.a — The 4^4 tile and slot space
A tile is the minimal storage surface of meaning inside the corpus graph. Each tile decomposes into slots by the product:
[
\mathsf{Slot} := \mathsf{Lens}\times\mathsf{Facet}\times\mathsf{Atom}
]
with:
(\mathsf{Lens} := {S,F,C,R}) with pinned order (S \prec F \prec C \prec R),
(\mathsf{Facet} := {1,2,3,4}) with numeric order,
(\mathsf{Atom} := {1,2,3,4}) with numeric order.
A tile slot is the triple ((L,f,a)\in\mathsf{Slot}). A tile instance binds a tile identifier (\mathsf{TileID}) to a finite set of slot payloads:
[
\mathsf{Tile}(\mathsf{TileID}) := {, (L,f,a) \mapsto \mathsf{PayloadRef}_{L,f,a},}.
]
Each payload reference is content-addressed by digest and schema ID:
[
\mathsf{PayloadRef} := \langle \mathsf{SchemaID},\ \mathsf{Digest},\ \mathsf{Size},\ \mathsf{Locator}\rangle.
]
C.S1.b — Addressing: GlobalAddr, SlotAddr, and LocalAddr
All semantics are addressable.
A GlobalAddr identifies a corpus object (chapter, appendix, registry object, or other globally routable node) and is parsed by AppA.
A SlotAddr refines a GlobalAddr by selecting a lens/facet/atom:
[
\mathsf{SlotAddr} := \langle \mathsf{GlobalAddr},\ L,\ f,\ a\rangle.
]
A LocalAddr refines a SlotAddr to an internal object path inside the payload schema:
[
\mathsf{LocalAddr} := \langle \mathsf{SlotAddr},\ \mathsf{FieldPath}\rangle.
]
LocalAddr is the atomic reference unit for local proofs, local link targets, and deterministic expansions.
C.S1.c — Formal address grammar and canonical ordering
Addresses MUST be parsed by AppA and canonicalized by AppA canonical bytes rules. Slot selection is canonical and must not depend on presentation.
Normative EBNF additions for SlotAddr and LocalAddr:
SlotAddr   := GlobalAddr "." Lens "." Facet "." Atom ;
LocalAddr  := SlotAddr ":" FieldPath ;

FieldPath  := PathSeg { "/" PathSeg } ;
PathSeg    := ID | ID "[" Int "]" ;

Canonical ordering of SlotAddr is the lexicographic order on:
[
(\mathsf{GlobalAddrCanon},\ L,\ f,\ a)
]
where (L) uses the pinned lens order and (\mathsf{GlobalAddrCanon}) is the AppA canonical form.
Canonical ordering of LocalAddr is lexicographic on:
[
(\mathsf{SlotAddrCanon},\ \mathsf{FieldPathCanon})
]
with FieldPathCanon computed by pinned normalization:
normalize . and .. segments are forbidden,
bracket indices are canonical integers (no leading zeros),
path separators are / only.

Within a tile, every payload schema MUST implement a minimal typed node vocabulary to support proof-carrying semantics.
Minimum local node types:
ClaimNode: a statement that can be true-typed under a corridor.
DefNode: definition node (introduces names, types, or constraints).
RuleNode: inference rule / rewrite rule node (typed, pinned).
DerivationNode: derivation
