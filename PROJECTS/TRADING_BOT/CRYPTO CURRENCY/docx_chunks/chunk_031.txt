N.C1.b — SalvageOutcome truth typing
Salvage outcomes are typed as:
OK: container mounts and verifies; no salvage required.
NEAR: partial recovery with bounded uncertainty; residual ledgers required.
FAIL: unrecoverable; must include minimal failure witnesses and may include an evidence plan if bounded recovery is possible.
CONFLICT: inconsistent recovery across replays; must emit conflict packet and fence.
Salvage truth typing must be deterministic under pinned salvage policies.

[

]

Redundancy mechanisms allowed (mirrors, erasure coding, witness reconstruction).

Which indices may be reconstructed from witnesses.
Whether reconstructed chunks are permitted; if yes, the new container must be a new content-addressed object, never altering the original digest.
If salvage produces a candidate container, it must be explicitly marked as a new container with new digest; the original remains unchanged.
N.C2.b — Salvage truth typing rules
NEAR requires ResidualLedgerRefs bounding what is uncertain (AppJ).
AMBIG requires EvidencePlanRef for further recovery steps (AppL).
FAIL requires a reproducible failure witness set.

N.C3 — Invariants
N.C3.a — No silent salvage
A salvaged container is never treated as the original; it is a new artifact. Any reliance on salvage must be disclosed via NEAR ledgers and salvage reports.
N.C3.b — Revocation precedence
Salvage cannot override revocation; if a chunk or key is revoked, salvage must treat it as blocked regardless of recoverability.

N.C4 — Protocols: Salvage regression fixtures
Salvage regression fixtures must cover:
Single-chunk corruption detection and failure witness.
Multi-chunk corruption with successful recovery via pinned redundancy.
Index corruption with reconstruction from witness bundles.
Pin manifest corruption: must fail or require evidence plan depending on policy.
Replay-index missing: must prevent deploy acceptance.
Conflict in reconstruction (two different recovered bytes): must emit conflict packet and fence.
Each fixture must bind expected salvage truth type and expected report digests.

O.F2 — ImportVerifier algorithm (mandatory steps)
Given bundle bytes (B):
Parse and canonicalize
Parse bundle into ExportBundle object and BundleManifest under CanonBytesDigest. Fail if parsing or canonicalization fails.
Verify internal digests
Run VerifyDigests; fail if any manifest/index/chunk digest mismatch.
Resolve schema closure
Load SchemaIndex and verify SchemaSetDigest closure is complete for all referenced objects. Reject if schema closure incomplete for deploy corridors; quarantine only if policy explicitly allows partial interpretability.
Resolve pins and pin closure
Load PinManifest; verify PinsDigest and ClosureDigest. Reject if missing required pins for ImportCorridorID.
Evaluate revocations and fences
Load RevocationIndex and FenceIndex. Compute effective revocation set and fence set under pinned propagation rules. If any required artifact is revoked, reject or quarantine per RevocationPolicy. Apply fences to routing/verification scope deterministically.
Fork rejection and registry anchoring
If bundle scope requires registry anchoring:
resolve RegistrySnapshotDigest,
verify RegistryCert and ForkPreventionCert in WitnessIndex,
compute fork set under ForkPolicyDigest,
reject/quarantine if forks detected per policy.
Corridor enforcement pre-check
Enforce static corridor constraints:
allowed edge kinds,
required artifacts present for MinTruth,
required disclosures present if NEAR is admissible.
Replay execution (mandatory where required)
Using ReplayIndex, execute required replay capsules under ReplayPolicyDigest:
verify harness digests,
verify environment pins,
execute deterministically,
compare output and transcript digests to expected digests.
If any required replay fails, reject/quarantine per policy.
Evaluate governance gates
Evaluate gate set; apply failure actions deterministically.
Compute final verdict
Combine outcomes deterministically using policy precedence:
digest mismatch ⇒ REJECT
revoked critical artifact ⇒ REJECT (unless policy allows quarantine)
replay failure on required capsule ⇒ QUARANTINE or REJECT per policy
gate failure ⇒ action per FailureActionSpec
otherwise ACCEPT
Emit import transcript and ImportAcceptanceCert
Import transcript is canonically serialized and content-addressed. ImportAcceptanceCert binds verdict, used policies, digests, and executed replays.
This algorithm is deterministic and must be replayable from bundle bytes and pinned verifier digest.
