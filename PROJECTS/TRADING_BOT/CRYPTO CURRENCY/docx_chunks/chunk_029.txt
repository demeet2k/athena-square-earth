K.S2.b — Conflict classification operator

K.S2.c — Fence application operator
Fence application is a deterministic function:
[
\mathsf{ApplyFence} : (\mathsf{FenceSpec}, \mathsf{GraphState}) \to \mathsf{MaskedGraphState}
]
Masking removes routability of:

excluded routes,
excluded artifacts,

Masking is overlay-only; the underlying objects remain present.
K.S2.d — Revocation propagation algorithm
Revocation propagation is deterministic and depends only on:
revocation receipts valid through pulse (p),
pinned propagation rules,
and dependency closure computation rules.
Let (\mathsf{DepClosure}(x)) compute the dependency closure of digest (x) under pinned rules.
For each effective revocation receipt (r) with target digest (d), define the revoked set:
[
\mathsf{RevSet}p := \bigcup{r:\ \mathsf{EffectivePulse}(r)\le p} \left({d}\cup \mathsf{Propagate}(d,r)\right)
]
where (\mathsf{Propagate}(d,r)) is defined by PropagationRuleDigest and may include:
forward closure: revoke all objects that depend on (d),
backward closure: revoke all objects (d) depends on (optional),
key-based closure: revoke all artifacts signed by compromised key digest (if TargetKind=Key),
profile closure: revoke profiles referencing revoked schemas/policies.
Propagation must be explicit and pinned. Unpinned propagation is illegal.

K.S3 — Invariants
K.S3.a — Minimal reproducibility invariant
A ConflictPacket must include sufficient information to reproduce the contradiction under pinned environment and pins. Absence of required reproduction inputs is a schema failure.
K.S3.b — Deterministic fence invariant
Given the same FenceSpec and same graph state digest, ApplyFence must produce identical masked-state digest. Any nondeterminism is CONFLICT.
K.S3.c — Revocation precedence invariant
Revocation overrides prior OK evidence for routing and import. If an artifact digest is in (\mathsf{RevSet}_p), then:
it is not admissible for OK routing,
it is not admissible for import acceptance under strict import policies,
any dependent claims must be downgraded via quarantine overlays.
K.S3.d — Append-only governance invariant
Conflicts, fences, quarantines, and revocations are appended as new objects and referenced via DeltaPacks (Ch20). No deletion or rewriting occurs.

K.S4 — Protocols
K.S4.a — ConflictPacket emission protocol
On detection of disagreement:
Confirm identical PinsDigest and EnvPinsDigest for the competing runs; if they differ, the disagreement is first typed as MIGRATE/PolicyMismatch, not CONFLICT.
Collect candidate witness objects and compute MinimalWitnessSet using pinned minimization order, within compute budget (\chi).
Construct ReproRecipe that:
references entry replay capsule(s),
binds closure digests for all required inputs,
defines deterministic step ordering.
Construct ExpectedDigests and ObservedDigests maps.
Construct FenceSpec:
include minimal exclusion targets necessary to prevent disputed routes from being used for OK.
Emit ConflictPacket with signatures.
Append ConflictPacket and FenceSpec via DeltaPack.
Update evidence tables (AppB) marking affected claims/routes as quarantined.
K.S4.b — Fence propagation protocol
For each FenceSpec:
Apply PropagationRuleDigest to compute the closure of excluded items:
edges directly excluded,
routes containing excluded edges,
claims whose only support is via excluded routes,
certificates that depend on excluded artifacts.
Emit a deterministic FenceExpansionReport artifact containing:
original targets digests,
expanded targets digests,
a proof trace digest for propagation.
Update the masked graph overlay state.
K.S4.c — Revocation issuance and import enforcement protocol
Revocation issuance:
Authority signs RevocationReceipt.
Receipt is appended via DeltaPack at IssuedPulse; EffectivePulse is enforced by policy.
Import enforcement:
Importer computes (\mathsf{RevSet}_p) for the pulse window it accepts (or for the current pulse under online acceptance).
If imported bundle contains any digest in (\mathsf{RevSet}_p), importer must:
reject, or
quarantine (only if policy allows), with explicit QuarantineRecord.
Importer must record the revocation receipts used (by digest) in the import transcript.
Revocation may require fallback deployment under Ch21 rules; fallback selection must be deterministic.

L.S3 — Invariants
L.S3.a — Boundedness invariant
EvidencePlan is valid only if:
candidates set is finite and ≤ MaxCandidates,
discriminators set is finite and ≤ MaxDiscriminators,
routes set is finite and ≤ MaxRoutes,
execution budget caps are finite and pinned.
Any unbounded search is forbidden.
L.S3.b — Determinism invariant

L.S3.c — Stop rule determinism invariant

L.S3.d — No silent promotion invariant
AMBIG cannot be promoted to NEAR/OK without emitting explicit promotion artifacts and evidence rows (AppB/Ch20). EvidencePlan must specify PromotionHooks; promotions must be append-only.
