A TL evaluation yields:

A MeasurementSpec defines what must be logged/emitted to:

15.6 RepairSpec (RS)
15.6.1 Definition
A RepairSpec defines the legal repair search space and legality constraints. A repair is a constrained, append-only transformation sequence.
RepairSpec is a pinned serializable record:

Where:
EdgeKind includes (minimum):
EQUIV (explicit equivalence edge),
DUAL (explicit dual interpretation edge),
MIGRATE (schema/profile migration edge),

Additional edge kinds require explicit pin extension.

For any claim of upgrade AMBIG→NEAR or NEAR→OK, a RepairCert must include the required witness bundle types and must reference the exact MS slices used.
15.6.2 Append-Only Doctrine and Boundary Constraints
All repairs are append-only:
No deletion of prior nodes/edges.
No mutation of prior boundary semantics.
Any meaning-changing transformation must be expressed as explicit edges with witnesses.
Carry/hatch boundaries (Ch14) are not errors. Repairs may only attach new structure on the later time axis; they may not “edit past” boundary semantics.

CE minimality is defined with respect to a deterministic partial order:
minimize involved_addresses cardinality,
then minimize evidence_slice record count,
then minimize evidence_slice byte size,
then minimize field set (drop optional fields while preserving reproduction),
tie-break lexicographically by canonical encodings.
A CE is certified minimal if the minimization algorithm exhausts all strictly smaller candidates under the partial order within budget and proves non-reproducibility for each.

15.8 Required Laws: Safety and Routability
The following laws are mandatory and enforce that defects are safe to route, replay, and govern.
15.8.1 Trigger Determinism Law
Under identical pins and identical inputs, TL evaluation is bitwise reproducible.
Formally, for any pinned state (S) and event stream (\mathcal{E}), define:
[
\mathrm{EvalTL}(S,\mathcal{E},\mathrm{TL}) \mapsto (\text{verdict}, \text{slice}, \text{trace_digest}, \text{obligations})
]
Then for any two executions (r_1,r_2) that both conform to pinned determinism constraints:
[
\mathrm{EvalTL}{r_1}(S,\mathcal{E},\mathrm{TL}) = \mathrm{EvalTL}{r_2}(S,\mathcal{E},\mathrm{TL})
]
byte-for-byte, including slice ordering and trace digest.
Enforcement. Any deviation is TLVerdict=FAIL and requires immediate fencing instructions in CE (or an interim fence artifact if CE is not yet formed).
15.8.2 Identifiability Law
A DS must be identifiable by a bounded measurement plan; otherwise it is retyped as AMBIG with an EvidencePlan obligation (AppL).
Let (\mathrm{Plan}(\mathrm{DS})) be the MS bounded plan associated with DSKind and pins. DS is identifiable if:
within MS bounds, the logged evidence determines the DSKind-specific witness payload and the DifferenceOperator outcome to the granularity required by corridor typing rules.
If not identifiable under bounds:
DSKind is set to CORRIDOR_TYPING_DEFECT with both sides AMBIG, or DS retains original DSKind but corridor class is forced to AMBIG_CORRIDOR and EvidencePlanRef is mandatory (pinned rule chooses which), and
TL must yield AMBIG when evaluation depends on missing evidence.
15.8.3 No Silent Repair Law
Any repair that changes meaning must be expressed as explicit edges (EQUIV/DUAL/MIGRATE/REF) with witnesses. Any attempt that cannot be witnessed must remain AMBIG or FAIL.
Let (\rho) be a repair sequence transforming artifact (A) to (A'). If the semantic interpretation class changes under pinned interpretation:
[
\llbracket A \rrbracket \neq \llbracket A' \rrbracket
]
then (\rho) must include at least one meaning-changing edge kind among {EQUIV, DUAL, MIGRATE, REF}, and the repair must emit witness bundles sufficient for verification.
If witnesses cannot be produced under RS constraints/budgets, the outcome is not upgraded; it remains AMBIG or FAIL, and CE is required when TL triggers.
15.8.4 Boundary Repair Discipline
Carry/hatch boundary events are legal structural boundaries. Repair may not edit past boundary semantics; it must attach new nodes/edges under append-only doctrine.
If a defect’s involved_addresses intersect a boundary clause region:
RS may only attach post-boundary edges/nodes,
any attempt to rewrite boundary semantics is illegal and results in RS rejection and CE obligation.

Defect-word / commutation isolation (operational defects).

For higher-order interaction/curvature defects, use the hypercube inclusion–exclusion extractor (\Delta_d) to isolate the pure (d)-way mode (Algorithm A.2).

15.9.2 Algorithm A.1: ShortestWordIsolator (Deterministic)
Goal. Find the canonical-minimal pair ((w_L,w_R)) of generator words producing a disagreement consistent with the DS difference operator.
Inputs.
generator set (G) (sorted by pinned order),
max_length (L_{\max}) from budgets,
canonical left form (C_L), right form (C_R),
difference operator (\mathrm{DiffOp}).
Output.
minimal ((w_L,w_R)) or NONE.
Procedure.
Enumerate word pairs in increasing total length (\ell = |w_L|+|w_R|) up to (2L_{\max}).
For each (\ell), enumerate pairs in pinned lexicographic order:
first by (|w_L|), then (|w_R|),
then lexicographically by word sequences (GeneratorID order).
For each candidate pair:
apply words via pinned kernel semantics:
[
X := w_L(C_L), \quad Y := w_R(C_R),
]
compute (\mathrm{DiffOp}(X,Y)) and test whether it matches the target diff class under pinned equivalence-of-diffs.
Return first match.
This procedure is deterministic by construction.

15.9.3 Algorithm A.2: Hypercube Inclusion–Exclusion Extractor (\Delta_d)
Purpose. Isolate pure (d)-way interaction terms with an optimal term count (2^d).
Let (d\ge 1). Let (U={1,\dots,d}). Let each axis (i) correspond to a binary choice of applying an operation or enabling a feature/interaction component (\theta_i \in {0,1}). Define a function:
[
F : {0,1}^d \to \mathcal{V}
]
where (\mathcal{V}) is the pinned difference algebra (vector-like: supports addition and additive inverses under pinned rules; if not available, the embedding into such an algebra is part of pins).
Define the hypercube difference operator:
[
\Delta_d[F] ;=; \sum_{S\subseteq U} (-1)^{d-|S|},F(\mathbf{1}_S),
]
where (\mathbf{1}_S\in{0,1}^d) has ones on indices in (S) and zeros elsewhere.
Operational instantiation.
(F(\mathbf{1}_S)) is the result (or a measured scalar/feature of the result) when exactly the subset of operations/features indexed by (S) is enabled/applied under pinned control profile.
The extractor yields the pure (d)-way interaction residual.
Optimality (term count). Under the pinned model where isolating the pure (d)-way multilinear coefficient requires distinguishing all (2^d) subset combinations, any extractor that is exact for all multilinear functions requires at least (2^d) evaluations. Therefore (\Delta_d) is optimal in term count under the standard evaluation oracle model. (The formal proof is pinned as part of AppA’s extractor library and referenced by extractor_proof_digest in control pins.)
Determinism.
Subset enumeration is canonical (increasing subset cardinality, then lexicographic).
Signs are computed by pinned parity rules.

15.9.4 Algorithm A.3: Deterministic Address Minimization
Goal. Minimize involved_addresses while preserving reproducibility of the defect class under replay.
Inputs.
initial address set (A_0) (sorted),
reproduction predicate (\mathcal{R}(A)): “using only addresses (A) and the MS evidence slice bounds, DiffOp outcome remains in target diff class,”
minimization budget (B).
Procedure (addresses-first delta minimization).
Initialize (A := A_0).
For each address (a) in (A) in canonical order:
propose (A' := A \setminus {a}),
if (\mathcal{R}(A')) holds within remaining budget, set (A := A').
Repeat pass until no removals occur or budget exhausted.
Output (A) and a minimization trace digest (hash of decisions in canonical order).
This procedure is deterministic and produces a canonical fixed point under the budgeted greedy minimization rule.

15.10 Construction (B): Trigger Evaluation

15.10.1 Algorithm B.0: EvalTL
Inputs.
pinned state (S),
event stream (\mathcal{E}) (as MS defines ordering and redaction),
TL.
Outputs.
[
(\text{verdict}, \text{slice}, \text{trace_digest}, \text{obligations})
]
Procedure.
Validate stream integrity.
Check MS schema conformance, ordering totality, redaction determinism markers, and digest chain validity.
If any check fails: verdict = FAIL; obligations include fence trigger key + integrity repair plan id.
Extract required measurements.
Collect the subset of fields/records referenced by TL.required_measurements.
If missing under bounds: verdict = AMBIG; obligations include EvidencePlanRef and missing key list digest.
Evaluate predicate_ir deterministically.
Evaluate TL.predicate_ir under pinned semantics.
Record an ordered trace of evaluation steps (operator, operands digests, result).
Compute trace_digest as digest of canonical encoding of the trace.
Compute minimal evidence slice used.
Apply TL.evidence_minimizer_ir to derive the minimal slice sufficient to reproduce the predicate outcome under pinned replay.
Minimality is defined by a pinned partial order (records-first then fields), and computed deterministically.
Emit obligations.
If verdict = AMBIG: obligations include missing evidence keys, suggested measurement upgrades allowed by MS bounds extension policy, and any AppL EvidencePlan hooks.
If verdict = FAIL: obligations include fences/quarantine key and remediation hooks.

15.12 Routing Discipline: From Defect to Governance
A defect artifact package is routable when the following are satisfied:
DS, TL, MS are pinned and have canonical digests.
EvalTL is deterministic (Trigger Determinism Law).
DS is identifiable under MS bounds, or explicitly AMBIG with EvidencePlan obligations.
No repair is silently applied; all meaning changes are explicit edges with witnesses.
Boundary repair discipline is enforced; illegal repair attempts immediately yield CE/fences.
If TL triggers and no OK/NEAR can be produced under RS budgets, CE is mandatory and must include fences/quarantine instructions.
The routing key for governance is the tuple of digests:
[
(\mathrm{ds_digest}, \mathrm{tl_digest}, \mathrm{ms_digest}, \mathrm{rs_digest?}, \mathrm{ce_digest?})
]
which uniquely identifies the mechanized failure instance under pins.
