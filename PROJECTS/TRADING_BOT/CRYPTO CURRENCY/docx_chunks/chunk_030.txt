❀ L.F — Flower: Discriminators as Typed Separators and Certificate-Gated Proof Paths
L.F1 — Objects
L.F1.a — Separator semantics
A discriminator partitions candidate space. For candidate set (C) and discriminator outcome (o), define:
[
C_o := {c\in C : \mathsf{Consistent}(c,o)}
]
where Consistent is pinned per discriminator.
Discriminators must be sufficient to eliminate at least one candidate on some outcome branch; discriminators that never change the candidate set are permitted only if required by policy (e.g., import verification prerequisite) and must be flagged as prerequisite discriminators.
L.F2 — Operators
L.F2.a — Certificate-gated discriminator execution
Some discriminators require certificates (ReplayCert, ImportCert, TransportCert). EvidencePlan must specify required certificates and reject outcomes that cannot be certified under pinned policies.
L.F3 — Invariants
L.F3.a — Outcome space completeness
For each discriminator, the outcome space is closed and pinned; unknown outcomes are FAIL.

M.S1.b — Meta-zero set and fixed corridor
Let (\mathsf{X}) be the joint state space on which Snap acts (typically a product of □○△ carriers and pinned derived observables). Let ({P_i}_{i=1}^m) be a pinned gate family, each enforcing one corridor constraint.

[

Snap is the canonical procedure to compute a representative near (\mathcal{Z}_\star) together with a residual ledger describing distances to each (Z_i) and any noncommutation effects between gates.

ResidualLedgerBundleRef is mandatory for NEAR. ReceiptBundleRef is mandatory when receipt checking is required by policy.

M.S2 — Operators
M.S2.a — Gate operator interface
Each gate (P_i) is a deterministic map:
[
P_i : \mathsf{X} \to \mathsf{X}
]
with:
gate digest,
metric digest for its residual,
domain spec digest (where the gate is defined),
and a canonical failure mode schema.
Each gate must provide a residual channel:
[
r_i(x) := d_i(x, P_i(x))
]
where (d_i) is pinned in ResidualSpec.
M.S2.b — Snap composite map
Given GateSeq ((P_1,\dots,P_m)), define:
[
T := P_m\circ \cdots \circ P_1.
]
M.S2.c — Snap iteration (with pinned relaxation)
Snap iteration is defined as a deterministic recurrence.
Strict iteration:
[
x^{(n+1)} := T(x^{(n)}).
]
Relaxed iteration (when RelaxSpec specifies (\lambda\in(0,1])):
[
x^{(n+1)} := (1-\lambda)x^{(n)} + \lambda,T(x^{(n)}),
]
where the linear combination is interpreted in the pinned carrier (\mathsf{X}) (if (\mathsf{X}) is non-linear, RelaxSpec must pin the interpolation operator; no implicit interpolation is permitted).
RelaxSpec must pin:
exact numeric representation policy,
rounding policy,
and whether (\lambda) is constant or schedule-dependent.
M.S2.d — StopSpec (deterministic stopping rules)
StopSpec defines a deterministic stop predicate:
[
\mathsf{Stop}(x^{(n)},x^{(n+1)},\mathbf{r}^{(n)},n,\mathsf{Budget})\in{\mathsf{true},\mathsf{false}}
]
where (\mathbf{r}^{(n)}=(r_1(x^{(n)}),\dots,r_m(x^{(n)}))). Required stop components:
MaxIter (bounded by (\chi)),
ConvergenceThreshold (pinned metric over state difference),
ResidualThresholds (pinned per-gate thresholds),
CycleDetect (deterministic detection of periodicity under pinned hash history window),
ReceiptGate (stop and fail/quarantine if required receipts fail).
Stopping must not depend on wall clock; it depends only on pinned values and iteration index.
M.S2.e — Residual ledger bundle emission
ResidualSpec pins the set of residual channels:
Minimum required channels:
Gate residuals (r_i(x)) for each gate.
Commutation residual(s) when gates are declared to commute or approximately commute:
for gate pairs ((P_i,P_j)) required by policy, compute:
[
c_{ij}(x) := d_{ij}(P_i(P_j(x)), P_j(P_i(x)))
]
with pinned metric (d_{ij}) and budget (\kappa).
Optional drift residuals measuring deviation from seed invariants or canonical normal forms (pins required).
Residuals must be emitted as a ResidualLedgerBundle:
[
\mathsf{RLB}:=
\langle
\mathsf{BundleVer},;
\mathsf{SubjectDigest},;
\mathsf{LedgerRefs},;
\mathsf{AggregateResidual},;
\mathsf{BudgetLedgerRef},;
\mathsf{PinsDigest}
\rangle
]
where LedgerRefs is an ordered list of ResidualLedgers (AppJ schema) keyed by channel.
M.S2.f — Snap function
The Snap operator is:
[
\mathsf{Snap} : (\mathsf{X},\mathsf{SnapPolicy},\mathsf{CorridorProfile}) \to \mathsf{SnapResult}
]
Algorithmically:
Initialize (x^{(0)}:=x).
For (n=0,1,\dots):
compute (x^{(n+1)}) by iteration rule,
compute residual channels,
verify receipts as required,
update budget ledger,
evaluate stop predicate.
On stop:
compute OutputStateDigest for (x^{(n_\star)}),
emit ResidualLedgerBundleRef,
emit required receipts bundle,
emit checkpoint,
emit replay capsule pointer,
type truth under corridor rules.
