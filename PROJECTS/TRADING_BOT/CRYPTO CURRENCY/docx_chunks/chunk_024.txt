C.S3 — Local Link Typing (REF/IMPL/PROOF/…) and Permission Rules
Local links are edges whose endpoints are LocalAddr or SlotAddr. Every edge is typed by LinkKind and is permitted only when the typing judgment holds.
C.S3.a — LinkKind vocabulary (minimum)
[
\mathsf{LinkKind} := {\mathsf{REF},\mathsf{IMPL},\mathsf{PROOF},\mathsf{TEST},\mathsf{DATA},\mathsf{CERT},\mathsf{EQUIV},\mathsf{DUAL},\mathsf{MIGRATE},\mathsf{CONFLICT}}
]
A tile MAY define additional link kinds only by schema migration; additional kinds must not be inferred.
C.S3.b — LocalEdge schema
[
\mathsf{LocalEdge} :=
\langle
\mathsf{EdgeID},;
\mathsf{From},;
\mathsf{Kind},;
\mathsf{To},;
\mathsf{PayloadRef?},;
\mathsf{PinsDigest},;
\mathsf{IssuedPulse}
\rangle
]
where:
(\mathsf{From},\mathsf{To}\in{\mathsf{LocalAddr},\mathsf{SlotAddr}}),
(\mathsf{PayloadRef?}) is required for kinds whose meaning depends on additional structure (PROOF/TEST/CERT/MIGRATE/CONFLICT),
(\mathsf{PinsDigest}) pins the interpretation closure,
(\mathsf{IssuedPulse}) binds the edge to time axis semantics (Ch20).
C.S3.c — Typing judgments
Let (\Gamma) be the local typing environment derived from pinned schema + pinned policy:
[
\Gamma := \langle \mathsf{SchemaPins},\ \mathsf{CorridorPins},\ \mathsf{PolicyPins},\ \mathsf{OperatorTablePins} \rangle.
]
Let (\Gamma \vdash u : \tau) denote that node/address (u) has type (\tau) under (\Gamma).
Edge permission is defined by judgments of the form:
[
\Gamma \vdash (u \xrightarrow{K} v)\ \mathsf{perm}
]
Minimum permission rules:
REF
[
\frac{\Gamma \vdash u:_* \quad \Gamma \vdash v:_*}{\Gamma \vdash (u \xrightarrow{\mathsf{REF}} v)\ \mathsf{perm}}
]
REF is allowed only as addressability; it carries no semantic commitment unless accompanied by PROOF/EQUIV/DUAL/MIGRATE.
IMPL (implementation link)
[
\frac{\Gamma \vdash u:\mathsf{SpecNode} \quad \Gamma \vdash v:\mathsf{CodeNode}}{\Gamma \vdash (u \xrightarrow{\mathsf{IMPL}} v)\ \mathsf{perm}}
]
IMPL payload MUST include:
code digest,
build pins,
interface contract digest,
and must be stable under corridor pins.
PROOF
[
\frac{\Gamma \vdash u:\mathsf{ClaimNode} \quad \Gamma \vdash v:\mathsf{LocalProofObject}}{\Gamma \vdash (u \xrightarrow{\mathsf{PROOF}} v)\ \mathsf{perm}}
]
PROOF payload MUST include:
proof object digest,
replay capsule digest (if applicable),
expected output digests,
and must pin the corridor spec used.
TEST
[
\frac{\Gamma \vdash u:\mathsf{ClaimNode} \quad \Gamma \vdash v:\mathsf{TestProof}}{\Gamma \vdash (u \xrightarrow{\mathsf{TEST}} v)\ \mathsf{perm}}
]
TEST payload MUST include:
fixture set digest,
harness digest,
pass/fail summary digest.
DATA
[
\frac{\Gamma \vdash u:\mathsf{ClaimNode} \quad \Gamma \vdash v:\mathsf{DataNode}}{\Gamma \vdash (u \xrightarrow{\mathsf{DATA}} v)\ \mathsf{perm}}
]
DATA is evidence attachment; its semantics are defined by Proof nodes that consume it.
CERT
[
\frac{\Gamma \vdash u:_* \quad \Gamma \vdash v:\mathsf{CertNode}}{\Gamma \vdash (u \xrightarrow{\mathsf{CERT}} v)\ \mathsf{perm}}
]
CERT payload MUST include certificate digest and verification policy pins.
EQUIV / DUAL / MIGRATE
These are meaning-bearing and require explicit witnesses:
[
\Gamma \vdash (u \xrightarrow{\mathsf{EQUIV}} v)\ \mathsf{perm}
\Rightarrow
\exists w:\mathsf{EquivWitness}\ \text{such that } u \xrightarrow{\mathsf{PROOF}} w \ \wedge\ v \xrightarrow{\mathsf{PROOF}} w.
]
Analogous requirements apply for DUAL and MIGRATE with corresponding witness types and corridor legality constraints. No EQUIV/DUAL/MIGRATE is permitted without witness payload references.
CONFLICT
CONFLICT links are permitted only to conflict packets:
[
\frac{\Gamma \vdash u:_* \quad \Gamma \vdash v:\mathsf{ConflictPacket}}{\Gamma \vdash (u \xrightarrow{\mathsf{CONFLICT}} v)\ \mathsf{perm}}
]
CONFLICT payload MUST include:
minimal witness set digests,
reproduction recipe digest,
fence spec digest.
C.S3.d — Forbidden inferences
No edge kind implies any other edge kind. In particular:
REF does not imply EQUIV.
IMPL does not imply correctness of implementation.
DATA does not imply validity.
Only PROOF/TEST/CERT evidence objects can upgrade truth status under AppI/AppM.

C.S4 — Invariants: Determinism, Local Closure, and Local Soundness
C.S4.a — Determinism of local expansions
For any fixed tile (\mathsf{Tile}), fixed rule ID (\mathsf{ExpandRuleID}), and fixed pins (\mathsf{Pins}):
[
\mathsf{did}(\mathsf{Expand}(\mathsf{Tile},\mathsf{ExpandRuleID},\mathsf{Pins}).\mathsf{Tile}')\ \text{is deterministic.}
]
If multiple expansions are possible without discriminating pins, the expansion is AMBIG and must emit an EvidencePlan (AppL) rather than selecting arbitrarily.
C.S4.b — Local closure invariant
A tile is locally closed iff:
all referenced digests resolve,
all referenced LocalAddr paths resolve within payload schemas,
all edge permission judgments hold under pinned typing environment (\Gamma).
Local closure failure is non-routable for OK and must be surfaced via typed failure nodes and, if exported, via container quarantine.
C.S4.c — Local proof soundness invariant
A ClaimNode may be treated as OK-routable only if:
there exists a PROOF or TEST edge to a witness object,
the witness verifies deterministically under pinned corridor spec,
and no applicable CONFLICT or revocation fences mask the claim.
C.S4.d — Canonical ordering invariants
All sequences committed to digests inside a tile MUST use pinned canonical ordering. Any unordered collection must be converted to an ordered sequence via pinned comparator prior to serialization/digesting.

D.S1.e — RegistryEntry schema
A RegistryEntry binds a typed key to a value under pinned rules.
[
\mathsf{RegEntry} :=
\langle
\mathsf{Key},;
\mathsf{Value},;
\mathsf{EntryType},;
\mathsf{PinsDigest},;
\mathsf{IssuedPulse},;
\mathsf{Supersedes?},;
\mathsf{Sig?}
\rangle
]
Key: typed identifier or typed composite key (e.g., (MsID, ProfileID), (SchemaID), (RouterID)).
Value: either a digest, a structured record digest, or a pointer record containing digest + retrieval info.
EntryType: enum describing semantics:
BIND (key binds to value),
ALIAS (key aliases another key),
MIGRATE (key transitions to new key/value with witness),
REVOKE (value revoked),
FENCE (quarantine/fence entry),
NOTE (non-semantic annotation; forbidden for interpretation-critical keys).
PinsDigest: pin closure required to interpret the entry (at minimum CanonBytesID + PolicyID).
IssuedPulse: Ch20 pulse binding.
Supersedes?: prior entry pointer for the same key (append-only replacement).
Sig?: signature if required by policy for the entry type.

D.S2 — Operators
D.S2.a — MsID derivation and deterministic collision resolution
Let (\mathsf{ManuscriptSeed}) be the canonical seed record for a manuscript:
[
\mathsf{ManuscriptSeed}:=\langle \mathsf{ManifestDigest},\ \mathsf{GenesisPulse},\ \mathsf{GenesisPolicyDigest},\ \mathsf{Issuer}\rangle
]
with canonical serialization.
Define the base MsID candidate:
[
\mathsf{MsID}0 := \mathrm{Trunc}{128}\big(\mathsf{H}(\textsf{"MsID"}\ |\ \mathsf{did}(\mathsf{ManuscriptSeed}))\big)
]
encoded as Ms⟨Hex⟩.
Collision resolution is deterministic:
Let (\mathsf{MsID}i := \mathrm{Trunc}{128}\big(\mathsf{H}(\textsf{"MsID/"}i\ |\ \mathsf{did}(\mathsf{ManuscriptSeed}))\big)) for (i\ge 1).
If (\mathsf{MsID}_0) is already bound to a different manuscript seed digest, choose the smallest (i\ge 1) such that (\mathsf{MsID}_i) is unbound or bound to the same seed digest. Record the collision resolution as a registry event with:
EntryType=NOTE forbidden; collision resolution must be BIND plus a CollisionRecord artifact signed under policy.
Any binding of a MsID must include the manuscript seed digest in its value payload to prevent silent forks.

[
\mathsf{ProfileID} := \mathrm{Trunc}_{128}\big(\mathsf{H}(\textsf{"ProfileID"}\ |\ \mathsf{did}(\mathsf{Profile}))\big)

where (\mathsf{Profile}) is serialized with ProfileID field zeroed. Collision handling follows the same deterministic suffix rule as MsID and must be recorded as a signed collision record if it occurs.

Define:
[
\mathsf{PolicyID} := \mathrm{Trunc}_{128}\big(\mathsf{H}(\textsf{"PolicyID"}\ |\ \mathsf{did}(\mathsf{PolicyPack}))\big)
