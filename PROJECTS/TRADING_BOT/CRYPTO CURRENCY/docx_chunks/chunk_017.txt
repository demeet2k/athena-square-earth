16.7 Required constructions (algorithms)
(A) Unified-step executor (reference algorithm)
Input: (S_{\text{total}}^{(0)}), step count (K), pinned profiles (resolved by pins), input stream ({x_k}).
Output: (S_{\text{total}}^{(K)}), checkpoint digests, transcript.
Pseudo-code (normative structure):
ExecUnified(S_total, K, Profiles, Inputs):
  AssertPinsPresentAndValid(S_total.Pi, Profiles)
  transcript = []
  checkpoints = []

  for k in 0..K-1:
    x = Inputs[k]  // may be empty but must be deterministic

    // AppN: normalize + seed
    xN = Canonicalize(x, Pi.pi_enc)
    seed = DeriveSeed(Pi.pi_rng, S_total.R, S_total.k, S_total.Pi)

    // AppO: ordered substeps
    B = EmptyBus(Pi)  // pinned schema/version
    for u in OrderFromPin(Pi.pi_order):
      if u == "CIRCLE":
        (S_total.C, B, logC) = CircleStep(S_total.C, Profiles.Circle, seed, xN, Views(S_total), B)
      if u == "TRIANGLE":
        (S_total.T, B, logT) = TriangleStep(S_total.T, Profiles.Triangle, seed, xN, Views(S_total), B)
      if u == "SQUARE":
        (S_total.Q, B, logQ) = SquareStep(S_total.Q, Profiles.Square, seed, xN, Views(S_total), B)

    // AppI: reconcile bus
    (S_total, B, logI) = ReconcileEndOfStep(S_total, B, Profiles)

    // AppM: advance + log + checkpoint
    S_total.k += 1
    entry = MakeTranscriptEntry(S_total.k, Pi, seed, B, logC, logT, logQ, logI)
    transcript.append(entry)

    if ShouldCheckpoint(S_total.k, Pi.pi_ckpt):
      D = CheckpointDigest(S_total, Pi.pi_enc, Pi.pi_H)
      checkpoints.append((S_total.k, D))

  finalD = FinalDigest(S_total, Pi.pi_enc, Pi.pi_H)
  return (S_total, checkpoints, finalD, transcript)

17.0 Normative Scope: Universal Action Grammar
This chapter defines the universal action grammar of HD‑SCT: every framework-permitted transformation SHALL be represented as a typed four-stage pipeline
[
\mathsf{P}\ \to\ \mathsf{Q}\ \to\ \mathsf{B}\ \to\ \mathsf{C},
]
with explicit pins, explicit stage state, and addressable artifacts at each stage. No transformation is well-formed unless it is fully placed into this spine with complete contracts and replay witnesses.
The spine is a compiler-grade contract system: it eliminates hidden degrees of freedom by requiring that all choice, phase, kernel selection, and corridor control enter only via declared injection points, represented as typed pins, and that all outputs are explicit addressable artifacts. Any transformation that cannot be uniquely assigned to one stage under the pinned interpretation rules SHALL be classified AMBIG and rejected from the deterministic lane unless explicitly routed through the ambiguity protocol in AppA/AppD.
17.1 Required Objects and Core Types
17.1.1 Canonical Serialization, Digests, and Addressing
Let (\mathsf{Ser}(\cdot)) be a canonical, deterministic, versioned serialization function (AppN) and (\mathsf{H}(\cdot)) be a cryptographic digest function.
Digest
[
\mathsf{Digest}(x) := \mathsf{H}(\mathsf{Ser}(x)).
]
Addressable artifact reference
[
\mathsf{ARef} := \langle \textsf{kind},\ \textsf{schema_id},\ \textsf{digest},\ \textsf{size},\ \textsf{media},\ \textsf{loc}\rangle
]
where (\textsf{digest}=\mathsf{Digest}(\textsf{payload})), and (\textsf{loc}) is a retrieval locator (content-addressed store, bundle offset, or import container pointer).
Every artifact emitted by any stage MUST be represented by an (\mathsf{ARef}). The payload MAY be embedded or external, but the digest MUST correspond to the canonical serialized payload.
17.1.2 Pins, PinSets, and Pin Schemas
A pin is the unique injection mechanism for all choice and control.
Pin
[
\mathsf{Pin} := \langle \textsf{pin_id},\ \textsf{type_id},\ \textsf{value},\ \textsf{value_digest}\rangle
]
with (\textsf{value_digest}=\mathsf{Digest}(\textsf{value})).
PinSet
[
\mathsf{PinSet} := \text{finite map }(\textsf{pin_id} \mapsto \mathsf{Pin})
]
with uniqueness of (\textsf{pin_id}).
Pin schema
[
\mathsf{PinSchema} := \langle \textsf{type_id},\ \textsf{constraints},\ \textsf{canonicalization}\rangle
]
governing validation and canonical form (AppN).
All nondeterminism MUST be eliminated by requiring that every potential tie, kernel choice, phase ordering decision, corridor routing, and any data-dependent branch that could vary across implementations be determined by pins. Any stage executor that reads ambient environment (time, host entropy, unspecified iteration order) without pin mediation violates determinism and SHALL fail certification.
17.1.3 Stage State and Stage Artifacts
A stage produces:
stage output value (typed data flowing to the next stage),
stage state (structured, explicit; never implicit prose),
stage artifacts (addressable, replay-critical).
Stage state MUST be serializable under (\mathsf{Ser}) and MUST have a digest. Stage state MAY be emitted as an artifact (recommended), but at minimum MUST appear as a structured object inside the stage transcript.
Define a uniform stage transcript record:
StageTranscript
[
\mathsf{StageTranscript} :=
\langle
\textsf{stage},\ \textsf{stage_spec_digest},\ \textsf{input_digests},\ \textsf{pins_used},\ \textsf{state_digest},\ \textsf{artifacts},\ \textsf{output_digest},\ \textsf{invariants_claimed}
\rangle.
]
Each transcript MUST list exactly which pins were consumed and MUST include digests for inputs, state, and outputs.
17.1.4 PipelineSpec ((\Pi)) as Typed Composition
A PipelineSpec is the minimal typed object defining a spine instance:
[
\Pi := \mathsf{C}\circ \mathsf{B}\circ \mathsf{Q}\circ \mathsf{P}.
]
Formally, a PipelineSpec is a 4‑tuple of stage specifications with explicit types:
StageSpec
[
\mathsf{StageSpec} :=
\langle
\textsf{stage},\ \textsf{version},\ \textsf{executor_id},\ \textsf{type_in},\ \textsf{type_out},\ \textsf{pin_requirements},\ \textsf{artifact_kinds},\ \textsf{invariants}
\rangle.
]
PipelineSpec
[
\mathsf{PipelineSpec} :=
\langle
\mathsf{P_spec},\ \mathsf{Q_spec},\ \mathsf{B_spec},\ \mathsf{C_spec},\ \textsf{compatibility_proof}
\rangle
]
where (\textsf{compatibility_proof}) attests that (\textsf{type_out}) matches (\textsf{type_in}) along the chain and that all required invariants are composable (ContractCert; AppC).
Stages MAY be identity stages, but identity MUST be explicit with a StageSpec and transcript; “skipping” a stage is represented as an identity stage with explicit invariants and empty artifact list.
