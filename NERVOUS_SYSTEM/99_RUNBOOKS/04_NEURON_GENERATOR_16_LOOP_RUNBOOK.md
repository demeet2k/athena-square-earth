# NEURON GENERATOR 16-LOOP RUNBOOK

## Purpose

This runbook operationalizes the official restart-aware loop protocol for full-corpus
deep synthesis.

## Standard Order

1. Read `95_MANIFESTS/GATE_STATUS.md`.
2. Attempt the live Docs gate with `Trading Bot/docs_search.py`.
3. Record the exact blocker or success result.
4. Read `95_MANIFESTS/NEURON_GENERATOR_STATE.md`.
5. Read the canonical cortex entry surfaces.
6. Read the strongest runtime-hub mirrors.
7. Choose the current `DeepLoop` frontier.
8. Ride one major metro line and one transfer hub.
9. Extract or refine neuron and synapse structure.
10. Land an improvement artifact.
11. Update ledger, manifest, queue, or edge surfaces.
12. Write the next restart seed.

## DeepLoop Actions

### Loop 1

- verify gate truth
- verify active surfaces
- state the admissible corpus scope

### Loop 2

- sweep `NERVOUS_SYSTEM/`
- identify strongest canonical maps, runbooks, and weak gates

### Loop 3

- sweep `self_actualize/mycelium_brain/`
- identify runtime prototypes worth canonical promotion

### Loop 4

- sweep `ECOSYSTEM/NERVOUS_SYSTEM/`
- identify governance definitions that should shape the canonical pass

### Loop 5

- ride the major metro lines
- record recurring transfer routes and zero-point hubs

### Loop 6

- ride transfer hubs
- record which hubs are overloaded, blocked, or underused

### Loop 7

- verify atlas and source-surface coverage
- identify replay gaps

### Loop 8

- inspect chapter and appendix contraction quality
- record which bridges to cortex remain weak

### Loop 9

- add or strengthen reusable neurons

### Loop 10

- add or strengthen synapses and edge records

### Loop 11

- preserve contradictions, blockers, and shadows
- do not smooth them away

### Loop 12

- build a concrete system improvement
- prefer schema, runbook, library, or bridge improvements

### Loop 13

- run replay-minded verification
- update validation queue if needed

### Loop 14

- enter the Fool checkpoint
- synthesize Chapter 11 and restart-token law
- inspect false completion
- trigger `CeremonialLoop -> 2`
- increment `ResetCount`

### Loop 15

- keep the preserved artifacts
- climb again with a sharper frontier

### Loop 16

- contract the run
- preserve a next seed
- expose unresolved gates explicitly

## Fool Reset Procedure

When `CeremonialLoop = 14`:

1. update `95_MANIFESTS/NEURON_GENERATOR_STATE.md`
2. set `fool_checkpoint.status: TRIGGERED`
3. set `ceremonial_loop.current: 2`
4. increment `reset_count`
5. keep `deep_loop.current` unchanged apart from the artifact-bearing increment
6. leave a short note explaining what visible certainty collapsed

## Minimum Deliverables

Every official run should leave:

- one synthesis ledger
- one loop-state manifest
- one neuron or synapse update
- one manifest or validation update

## Failure Modes

- visible restart without preserved artifacts
- deep progress reset when only ceremonial progress should reset
- no writeback to manifests
- no neuron-level improvement
- claiming full-corpus coverage without atlas or hub evidence
- claiming live Docs evidence while blocked
