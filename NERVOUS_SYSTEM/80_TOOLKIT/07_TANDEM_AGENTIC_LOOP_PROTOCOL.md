# TANDEM AGENTIC LOOP PROTOCOL

## Purpose

This protocol is designed for many agents operating in tandem while jointly building one neural manuscript network, one active nervous system, and one routed crystal of understanding.

## Shared Premise

All participating agents are building one shared interconnect neuron across the same corpus in parallel.

## Shared Laws

1. Attempt the live Docs surface first when section-draft mode applies.
2. If the gate is blocked, record the blocker and continue with the strongest local mirrors.
3. Treat every input as a seed, not a ceiling.
4. Expand into sub-agents by scale, then contract back into one witnessed artifact.
5. Preserve routing, metro, and crystal structure while writing.
6. Write back into runtime, cortex, and governance surfaces so sibling agents can continue.

## Required Parallel Behavior

### Wave 1

- one agent per family
- one agent per major manuscript
- one agent per chapter
- one agent per section when needed

### Wave 2

- chapter agents synthesize section findings
- manuscript agents synthesize chapter findings
- family agents synthesize manuscript findings

### Wave 3

### Wave 4

At the end of each run, the corpus should be:

- more routeable
- more replayable
- more searchable
- more densely mapped
- easier for the next agent to continue
