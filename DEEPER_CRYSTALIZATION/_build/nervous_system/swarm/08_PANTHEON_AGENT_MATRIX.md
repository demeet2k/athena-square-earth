# PANTHEON AGENT MATRIX

## Purpose

The swarm cannot be a pile of identical workers. It must assign stable roles so parallel passes do not collapse into duplicated effort. The pantheon matrix defines the 16 level-two agent identities, then shows how they expand to 64 and 256 lawful subagents.

## The Four Pillars

The diagonal agents are pillars because they preserve one element through both parent and child positions.

### EE - Ledger Keeper
- owns boundaries, constraints, invariant checks, contradiction prevention
- primary outputs: ledgers, caps, stop conditions, proof boundaries

### WW - Continuity Weaver
- owns continuity across passes, narrative coherence, family memory, transition smoothness
- primary outputs: continuity notes, family summaries, roll-forward state

### FF - Frontier Igniter
- owns expansion, bold hypothesis, new line creation, source ignition
- primary outputs: candidate families, new routes, missing-node hypotheses

### AA - Cartographic Architect
- owns maps, schemas, address systems, routing structure, compression laws
- primary outputs: tensors, metro maps, addressing forms, topology docs

## The Twelve Archetypes

Ordered mixed pairs matter because parent-to-child direction preserves bias.

### EW - Grounded Synthesizer
- grounds flowing manuscript material into stable form
- best for: canonical markdown promotion from raw source text

### EF - Constraint Breaker
- tests where current boundaries are too rigid
- best for: detecting underfitting, stale summaries, artificial caps

### EA - Structural Custodian
- preserves architecture while enforcing feasibility
- best for: map updates that must remain replay-safe

### WE - Intake Empath
- receives rough material without premature collapse
- best for: manuscript ingestion and ambiguity-preserving intake

### WF - Narrative Amplifier
- expands live narrative bodies into richer linked families
- best for: family synthesis, chapter-to-chapter threading, story lines

### WA - Relay Translator
- moves fluid text into navigable structures
- best for: section maps, headings, handoff protocols, registry conversion

### FE - Experimental Driver
- pushes the system against constraint boundaries to reveal the next lever
- best for: frontier selection and dissatisfaction escalation

### FW - Catalyst Integrator
- expands by synthesis instead of fragmentation
- best for: cross-family merging and emergent line discovery

### FA - Formal Exploder
- turns abstract law into new structural variants
- best for: new routing schemes, tensor extensions, algorithm generation

### AE - Atlas Cartographer
- maps concrete evidence into address space
- best for: atlas grounding, topological counts, live body maps

### AW - Transport Mediator
- keeps meaning moving across bodies without loss
- best for: hub assignment, lane transfers, doc-to-md promotion

### AF - Theorem Compiler
- converts abstract structures into implementable control logic
- best for: protocols, pseudocode, runtime docs, verification harnesses

## 16 -> 64 Expansion Law

Every pillar or archetype splits into four level-three submodes:
- E submode: verify and bound
- W submode: merge and compare
- F submode: expand and perturb
- A submode: map and abstract

Therefore:
- 16 level-two roles x 4 level-three submodes = 64 task cells

Example:
- `AE-E`: atlas cartographer in verification mode
- `WF-F`: narrative amplifier in expansion mode
- `EA-A`: structural custodian in mapping mode

## 64 -> 256 Expansion Law

Every level-three cell splits into four level-four output atoms:
- witness
- route
- delta
- next

Therefore:
- 64 task cells x 4 atoms = 256 swarm outputs

Meaning of the atom classes:
- witness: what concrete source or file justified the move
- route: where the move belongs in hubs, rails, scale, and family space
- delta: what changed in the nervous system
- next: what the next recursive pass should do

## Address Law

Use the first two lineage positions for the pantheon role and the last two lineage positions for submode and atom:

`SwarmAddr = <G1G2, G3, G4>`

Examples:
- `AE-E-witness`
- `FW-A-next`
- `EE-E-delta`

This address law is compatible with the existing 4x4 lineage while making role ownership explicit.

## Deployment Rule

When a pass is parallelized:
1. one pillar must own invariants
2. one archetype must own atlas grounding
3. one archetype must own frontier expansion
4. one archetype must own routing compression

Minimum viable swarm quartet:
- `EE`
- `AE`
- `FE`
- `AW`

## Current Recommended Swarm for the Active Frontier

## Failure Law

If multiple agents produce the same artifact class without a declared ownership split, the swarm is not actually parallel. It is duplicating labor. Pantheon ownership exists to prevent that collapse.
