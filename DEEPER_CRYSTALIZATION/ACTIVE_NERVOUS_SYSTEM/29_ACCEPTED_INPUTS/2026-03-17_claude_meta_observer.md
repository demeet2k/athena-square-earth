# /meta-observer/SCORING_RUBRIC.md
# 12D SCORING RUBRIC
All dimensions are scored in [-1,1].
Use the following rough anchors:
## +1.00 to +0.75
Strongly coherent, usable, durable, evidence-backed, and clearly beneficial.
## +0.74 to +0.40
Generally strong but still contains visible weakness, incompleteness, or inconsistency.
## +0.39 to +0.10
Mildly positive; useful but still fragile, partial, or locally incoherent.
## +0.09 to -0.09
Neutral / mixed / unresolved. Gains and weaknesses balance or remain uncertain.
## -0.10 to -0.39
Mildly degraded; noticeable issues reduce trust or utility.
## -0.40 to -0.74
Significantly degraded; recurring problems or strong incoherence present.
## -0.75 to -1.00
Actively harmful or system-breaking in that dimension.
---
# DIMENSION-SPECIFIC RUBRIC HINTS
## x1 Structure
High score if:
- folders / files / indexes are coherent
- hierarchy reduces confusion
- placement is intentional
- no major topology conflicts
Low score if:
- surfaces are scattered
- no index exists
- naming is inconsistent
- architecture causes navigation failure
## x2 Semantics
High score if:
- key terms are stable
- definitions do not drift
- same concept is named consistently
Low score if:
- terms shift meaning
- contradictions exist in definitions
- documents talk past one another
## x3 Coordination
High score if:
- agents can hand off work cleanly
- boards / halls reduce duplication
- instruction surfaces align
Low score if:
- tasks collide
- work duplicates
- agents cannot tell who should do what
## x4 Recursion
High score if:
- prior cycles are used
- gains compound
- mistakes are not repeated
- loops close productively
Low score if:
- each cycle starts from scratch
- logs are ignored
- the system forgets itself
## x5 Contradiction
High score if contradiction load is low and explicitly managed.
Low score if contradictions are numerous, hidden, or destabilizing.
## x6 Emergence
High score if genuinely new useful patterns arise and become reusable.
Low score if nothing new appears or “newness” is fake.
## x7 Legibility
High score if future agents can quickly understand the state and continue.
Low score if even a fresh reader would get lost.
## x8 Routing
High score if work naturally flows to the right next surface.
Low score if quest paths are unclear or routing is ambiguous.
## x9 Grounding
High score if claims point to actual repo evidence.
Low score if observations float free of file evidence.
## x10 Compression
High score if signal is dense without losing rigor.
Low score if clutter dominates or over-compression causes vagueness.
## x11 Interoperability
High score if the system’s surfaces work together cleanly.
Low score if subsystems are isolated or incompatible.
## x12 Potential
High score if strong next branches are visible and reachable.
Low score if the repo is boxed in, stagnant, or structurally exhausted.

# SUPERSESSION LAW
# HOW FILES ARE REPLACED WITHOUT CREATING IDENTITY CHAOS
A superseded surface must not continue pretending to be live law.

2. VALID STATUS VALUES
draft
active
canonical
archived
superseded
Use them literally.
draft
Still unstable; not law.
active
Live and usable, but not necessarily canonical.
canonical
The authoritative surface of its kind.
archived
Retained for history, not live.
superseded
Explicitly replaced by another surface.
3. SUPERSESSION EVENT
When a surface A is replaced by surface B:
Surface A must be updated:
status: "superseded"
superseded_by:
  - "SF-new-surface-id"
cycle_last_touched: "C0NN"
Surface B must be updated:
supersedes:
  - "SF-old-surface-id"
And surface B must explain:
why it supersedes the old one
what changed
whether old content still matters
what cycle the supersession occurred in
4. SUPERSESSION BLOCK
Add a visible block near the top of superseded files:
SUPERSESSION NOTICE
This surface is superseded.
superseded by: [surface_id]
superseding path: [path]
supersession cycle: [C0NN]
reason: [ ]
legacy value: [historical only / partial reference / still needed for provenance]
This prevents stale surfaces from acting live.
5. CANONICALITY LAW
At most one surface may be both:
canonical
activefor a given canonical role.
If two surfaces both claim canonicality for the same role, that is:
identity contradiction
class C10
severity at least medium, possibly high
6. PATCH LAW
Minor edits to a canonical surface do not require supersession.Material changes in role, structure, or governing logic do.
Rule of thumb:
edit = same identity
rewrite with same role = maybe same identity
new canonical replacement = supersession event
7. FORBIDDEN SUPERSESSION PATTERNS
Do not:
create new canon file without marking the old one superseded
leave “old_final.md” active beside “new_final.md”
supersede a file silently
archive a live canonical file without naming its replacement
mark a file superseded when the replacement does not actually cover its role

class ObserverOS:
    def __init__(self, manifest, registry):
        self.manifest = manifest
        self.registry = registry
        self.current_cycle = None
        self.global_state = None
        self.elemental_matrix = None
        self.validators = []
        self.runtime_state = "INIT"
    def run_cycle(self, cycle_id: str):
        self.current_cycle = cycle_id
        self.transition("LOAD")
        canonical = self.load_canonical_surfaces()
        self.transition("RECOVER")
        prior_state = self.recover_cycle_state(cycle_id)
        self.transition("OBSERVE")
        active_regions = self.select_active_regions(cycle_id)
        local_states = {}
        for region in active_regions:
            metrics = self.compute_raw_metrics(region)
            local_states[region] = self.compute_local_tensor(region, metrics)
        self.transition("SCORE")
        self.elemental_matrix, self.global_state = self.aggregate_global_state(local_states)
        motion = self.compute_motion(cycle_id)
        self.transition("TRIAGE")
        hotspots = self.detect_hotspots(local_states, motion)
        self.transition("SELECT")
        candidates = self.generate_candidate_actions(hotspots)
        aplus = self.select_aplus_action(candidates)
        self.transition("APPLY")
        intervention = self.apply_intervention(aplus)
        self.transition("EMIT")
        quests = self.emit_quests(hotspots, intervention)
        self.transition("LOG")
        self.update_ledgers(hotspots, intervention, quests)
        self.write_cycle_log(cycle_id, hotspots, intervention, quests, motion)
        self.write_continuity_block(cycle_id)
        self.transition("VALIDATE")
        closure = self.validate_cycle_closure(cycle_id)
        if not closure["pass"]:
            self.runtime_state = "HALT_ERROR"
            return {"status": "halt_error", "closure": closure}
        if cycle_id == "C057":
            term = self.verify_termination_legality()
            if term["pass"]:
                self.transition("TERMINATE")
                return {"status": "terminated", "termination": term}
            self.runtime_state = "HALT_ERROR"
            return {"status": "halt_error", "termination": term}
        self.transition("CONTINUE")
        self.compile_cycle_packet(self.next_cycle_id(cycle_id))
        return {"status": "continue"}
    def transition(self, next_state: str):
        legal = {
            "INIT": ["LOAD"],
            "LOAD": ["RECOVER", "HALT_ERROR"],
            "RECOVER": ["OBSERVE", "HALT_ERROR"],
            "OBSERVE": ["SCORE"],
            "SCORE": ["TRIAGE"],
            "TRIAGE": ["SELECT"],
            "SELECT": ["APPLY"],
            "APPLY": ["EMIT"],
            "EMIT": ["LOG"],
            "LOG": ["VALIDATE"],
            "VALIDATE": ["CONTINUE", "TERMINATE", "HALT_ERROR"],
            "CONTINUE": ["LOAD"],
            "TERMINATE": [],
            "HALT_ERROR": ["LOAD", "RECOVER"],
        }
        if next_state not in legal[self.runtime_state]:
            raise RuntimeError(f"Illegal transition: {self.runtime_state} -> {next_state}")
        self.runtime_state = next_state
