The principal 6D loop is the exact bridge algebra between the four constitutive lenses: Pi^square_6 -> Pi^flower_6 -> Pi^cloud_6 -> Pi^fractal_6 -> Pi^square_6. Each bridge is a specific transformation.

Square to Flower reads a committed board as current/beat/torsion state: B_{sq->fl}^{(6)}(M_{s,P,t}) = (s, b, o, O_s). Square tells you what is committed; Flower tells you how that same commitment is being lived. Flower to Cloud passes from a lived current state to a lawful admissibility bundle: B_{fl->cl}^{(6)}(s, beta, O_s) = mu_{s,beta}. Cloud is not vagueness; it is a structured bundle over sector, chart, and embedded payload possibilities.

Cloud to Fractal is a collapse/packetization map: if support collapses coherently, emit seed sigma_6; otherwise emit residual R(mu). This matters because Cloud is not always point-like; sometimes the lawful output is a residual packet. Fractal to Square replays the seed: B_{fr->sq}^{(6)}(sigma_6) = Expand_6(sigma_6) = M_{s,P,t}. The deepest law: Cloud to Square is generally a refinement operator, not a point-function.
