## Canonical Object Family

The v1 object family is frozen as follows.

- `SemanticEmbassySurface`: the whole typed reception organ after Release Notary.
- `ClaimSurface`: the attested meaning, claim class, ceiling, support refs, replay refs, and forbidden-claim ceiling.
- `AudienceProfile`: the target audience, burden tolerance, vocabulary constraints, prior context, and trust sensitivity.
- `RegisterProfile`: the chosen export register: `minimal`, `diplomatic`, or `rhetorical`.
- `InterfaceContract`: the typed agreement stating which audience and register pair are admissible.
- `MessageEnvelope`: the outer carrier that moves the rendered claim through one named public shell.
- `TranslationSurface`: the concrete rendered wording for a chosen interface.
- `DomainPack`: the allowed domain vocabulary, blocked vocabulary, benchmark-safe names, and federation-safe terms.
- `ReleaseBundle`: the signed release object that binds claim ceilings to a release-trust lineage.
- `SafeWordingBundle`: the allowed phrases, blocked phrases, downgrade replacements, benchmark-safe names, and federation-safe names.
- `KairosAssessment`: the timing judgment over whether to emit now, delay, or withhold.
- `RhetoricalPreview`: the preview-only record of ethos, pathos, logos, catharsis mode, and style pressure.
- `SemanticExportValidatorResult`: the pass/fail verdict proving that the rendered surface stayed inside the allowed ceiling.

## Frozen Dataflow

The v1 dataflow is fixed as:

`RecertificationPack/ReleaseTrustPack -> ClaimSurface -> AudienceProfile + RegisterProfile -> SafeWordingBundle -> TranslationSurface -> SemanticExportValidatorResult -> AudienceFacingBundle`

This flow is directional for a reason. Meaning is not allowed to start from audience appetite and work backward into inflated truth. The attested claim comes first. Audience adaptation is lawful only after ceiling and support are already fixed.

## Render Policy Map

The release-tier to render-mode map is frozen in v1:

- `explanatory -> minimal`
- `benchmark_safe -> minimal`
- `release_safe -> diplomatic`
- `public_claim -> diplomatic`
- `federation_claim -> diplomatic` with federation-safe terminology required
- `rhetorical -> preview_only` unless a later wave explicitly legalizes external/public rhetoric

The register set remains:

- `minimal`: schema-close wording, low affect, low metaphor, benchmark-safe naming when needed
- `diplomatic`: audience-aware language that preserves the claim ceiling without inflating it
- `rhetorical`: expressive and affect-bearing language, available only as preview in v1

## Validator Law

The Semantic Export Validator is the central governor of this organ. In v1 it must reject:

- any rendered wording that exceeds the attested claim ceiling
- any rhetorical/public output request
- any surface missing replay refs or support refs
- any bundle missing downgrade replacements
- any audience/register pairing outside the declared policy
- any benchmark-facing export that lacks benchmark-safe naming
- any federation-facing export that lacks federation-safe terminology

The validator is not ornamental. It is the burden-preserving membrane that stops audience optimization from turning into overclaim.
