### Chapter thesis

Equations do not arise first as symbols. They arise as conserved morphology over typed
source graphs. This chapter therefore refuses the common mistake of imposing elegant
formalism onto weakly normalized material before the manuscript has earned the right to
do so. It begins instead with source objects, promotes them into graph-bearing
relations, stabilizes recurring morphological signatures, tests those signatures under
rewrite, and only then allows equation-like objects to appear as compressed invariants.

The chapter is the law of graphal pre-equation structure. If earlier chapters define
the admissibility conditions under which a manuscript can speak, then Chapter 8
defines the moment the manuscript's material becomes anatomically legible. A document,
corpus shard, or source family stops being a loose textual surface and becomes a typed
relational skeleton. Only after that skeleton has shown replayable structural

### 8.1 Square - Source graphs and morphological objects

#### 8.1.1 Source objects and typed source graphs

A source object is any bounded, origin-bearing artifact or artifact slice from which
lawful graph structure may be extracted. In this chapter, source is not merely
evidence. It is graph-bearing material. A source object must therefore be typed at a
higher granularity than ordinary citation handling requires.

Define a source object as

`s = (a_s, k_s, o_s, p_s, m_s, mu_s, ell_s, gamma_s)`

where:

- `a_s` is the source address
- `k_s` is the source class
- `o_s` is the origin descriptor
- `p_s` is the provenance record
- `m_s` is the modality profile
- `mu_s` is the uncertainty and source-quality profile
- `ell_s` is the lineage
- `gamma_s` is the governance state

From one or more such sources the chapter induces a typed source graph

`G_s = (V_s, E_s, kappa_V, kappa_E, I_s, ell_s)`

where:

- `V_s` is the node family
- `E_s` is the edge family
- `kappa_V` is the node-kind typing
- `kappa_E` is the edge-kind typing
- `I_s` is the current invariant family
- `ell_s` is the graph lineage

The decisive restriction is that not every adjacency, co-occurrence, or lexical echo
is graph truth. A candidate edge must be typed, provenance-bearing, and later
replayable before it can be admitted into the lawful graph.

#### 8.1.2 Morphological signatures

A morphology is a stable structural shape over a graph region. It is not a single
surface resemblance but a typed family of organization capable of surviving lawful
normalization, rewrite, or transport.

Define a morphological signature as

`mu_G = (a_mu, chi_mu, V_mu, E_mu, iota_mu, x_mu, ell_mu)`

where:

- `a_mu` is the signature address
- `chi_mu` is the target scope
- `V_mu` and `E_mu` are the local node and edge subfamilies
- `iota_mu` is the invariant core
- `x_mu` is the exception ledger
- `ell_mu` is the lineage

A morphology is lawful only when a nontrivial invariant core persists under a declared
rewrite family. One-off resemblance, visual neatness, or isolated repetition do not
suffice.

Canonical morphology classes include:

`K_M = {Chain, Cycle, Hub, Bridge, Fork, Merge, Lattice, Shell, Spine, Mirror}`

These are not yet equations. They are the structural atoms from which equation-like
law later emerges.

#### 8.1.3 Equivalence, bounded similarity, and structural divergence

Chapter 8 must decide when two graph regions are genuinely equivalent, merely similar,
or fundamentally distinct. Let `A` and `B` be typed graph regions. Define

`A ~_morph B`

if and only if:

1. node and edge classes align
2. invariant cores align
3. rewrite-normal forms align inside the declared corridor
4. remaining differences belong only to the exception ledger

Bounded similarity is written

`A ~=_morph B`

and true divergence is written

`A !~_morph B`

This three-way separation is one of the chapter's central services to Chapter 9. A
universal transition calculus cannot be trusted if Chapter 8 has already collapsed
deep sameness, bounded analogy, and false symmetry into one bucket.

#### 8.1.4 Equation genesis from invariant preservation

The chapter's strongest Square claim is that an equation candidate is a symbolic
compression of a graph or morphology law. It is not the first law itself. Let

`R_rew = {r_1, ..., r_n}`

be a rewrite family acting on graph regions. If a stable quantity, relation, or
topological class is preserved across that family, the manuscript may compress that
preservation into an equation-like object:

`e = Compress(I_preserved(R_rew, G_s))`

The generative chain is therefore:

`source -> graph -> morphology -> invariant -> equation candidate`

This is the exact pre-CUT law of the chapter.

### 8.2 Flower - Graph growth, rewrite, and equation formation

#### 8.2.1 Ingestion from source to graph

The first dynamic process of the chapter is lawful graph formation:

`Ingest_graph : S_src -> G_src`

This ingestion includes segmentation of source artifacts, proposal of candidate nodes,
typed edge construction, provenance attachment, uncertainty tagging, and preliminary
local invariant detection. The graph therefore begins not as a perfect frozen object
but as a living structure already more disciplined than an unstructured corpus slice.

#### 8.2.2 Morphological normalization

The second dynamic is normalization:

`Norm_morph : G_s -> G_hat_s`

where `G_hat_s` is the morphology-normalized graph. Normalization may prune low-support
edges, collapse merely lexical variants, lift recurrent local subgraphs into motif
classes, and align graph orientation or role labeling. This is not arbitrary
simplification. It is a replay-safe move toward structural comparability.

#### 8.2.3 Rewrite families and transform law

Morphology becomes mathematically meaningful only when tested under lawful rewrites. If
`r in R_rew` is a rewrite operator, then a graph region `A` transforms as

`A --r--> A'`

The chapter is interested in rewrite families that preserve something. If an invariant
`iota` satisfies

`iota(A) = iota(A')`

for all rewrites in a meaningful family, then that invariant becomes candidate
substrate for equation synthesis. The transition from morphology to equation is
therefore dynamic rather than purely visual.

#### 8.2.4 Equation-candidate synthesis

Once enough rewrite-preserved invariants have been found, the manuscript may synthesize
equation candidates:

`Synth_eq : (R_rew, I_pres) -> E_eq`

Every candidate must preserve explicit reference to:

- the graph region or motif it compresses
- the invariant family
- the rewrite family under which the invariant holds
- the exception ledger
- the burden class under which the candidate is asserted

A candidate that hides all structural exceptions is not lawful elegance. It is early
overclosure.

#### 8.2.5 Local-to-chapter synthesis

The chapter must support not only one graph and not only one local invariant, but
whole families of them. Define local-to-chapter synthesis as

`Synth_ch : {mu_G_i}_{i=1}^n -> E_ch`

where `E_ch` is the chapter-level morphology and equation crystal. This is the stage
where Chapter 8 prepares the larger transport into Chapter 9's universal transition
law.

### 8.3 Cloud - Ambiguity, weak provenance, and false symmetry

#### 8.3.1 Weak-provenance graph material

Many graph regions arise from weak, partial, or mixed-provenance material. The chapter
must preserve that weakness explicitly rather than laundering it into later exactness.
Define weak-provenance graph material as

`G_s^partial = (V_s, E_s, mu_s, p_s)`

Such material may still support exploratory morphology generation, benchmark design,
and provisional equation candidates, but it cannot be promoted to high-burden closure
without more evidence.

#### 8.3.2 Morphological ambiguity bundles

Different graph regions may support more than one lawful morphological reading. The
chapter therefore defines an ambiguity bundle

`B_morph = (M_cand, E_need, R_replay, X_exceptions)`

where:

- `M_cand` is the candidate morphology family
- `E_need` is the evidence still needed
- `R_replay` is the replay or reread family
- `X_exceptions` is the incompatibility ledger

This bundle is a lawful output. The chapter rejects the pressure to collapse every
pattern into one canonical reading too early.

#### 8.3.3 False symmetry and chorus morphology

One of the chapter's main risks is false symmetry: graph regions may look similar
because they inherit the same weak convention, lexical repetition, or projection bias,
not because they share a real invariant structure. Define symmetry risk as

`H_sym(A,B) = h_1 * not IndependentSource(A,B) + h_2 * LexicalOnlyMatch(A,B) + h_3 * not RewriteStable(A,B)`

When `H_sym` is high, equivalence promotion must be withheld. This is the graph-scale
analogue of later warnings against false resonance in multi-agent or field-level
synthesis.

#### 8.3.4 Equation overcompression

The chapter must also guard against equation theater: the emission of elegant symbolic
forms that erase the exception family, burden boundaries, or provenance limits of the
material that generated them. Define overcompression by

`Overcompress(e) iff ExceptionSurface(e) < TrueExceptionSurface(G_s, R_rew)`

Any lawful equation candidate must remain coupled to source graph lineage, rewrite
family, preserved invariant, exception ledger, and corridor of applicability.

#### 8.3.5 Formal ignorance

Sometimes the system knows that a graph is structured but does not yet know the right
morphology class. Sometimes it knows an invariant exists but has not yet earned a clean
equation candidate. In both cases Chapter 8 permits formal ignorance. This protects
later CUT stages from inheriting fake exactness purchased by premature symbolic
compression.

### 8.5 Canonical algorithmic form

```python
from dataclasses import dataclass, field
from typing import Dict, List, Any

@dataclass
class SourceNode:
    address: str
    node_kind: str
    payload: Dict[str, Any]
    provenance: Dict[str, Any]
    lineage: List[str] = field(default_factory=list)

@dataclass
class SourceEdge:
    address: str
    edge_kind: str
    src: str
    dst: str
    uncertainty: Dict[str, Any]
    lineage: List[str] = field(default_factory=list)

@dataclass
class MorphSignature:
    address: str
    node_refs: List[str]
    edge_refs: List[str]
    invariant_core: Dict[str, Any]
    exception_ledger: List[str]
    lineage: List[str] = field(default_factory=list)

def build_source_graph(nodes: List[SourceNode], edges: List[SourceEdge]) -> Dict[str, Any]:
    return {
        "node_refs": [n.address for n in nodes],
        "edge_refs": [e.address for e in edges],
        "lineage": [n.address for n in nodes] + [e.address for e in edges],
    }

def extract_morph_signature(graph: Dict[str, Any]) -> MorphSignature:
    return MorphSignature(
        address="morph/auto",
        node_refs=list(graph["node_refs"]),
        edge_refs=list(graph["edge_refs"]),
        invariant_core={},
        exception_ledger=[],
        lineage=list(graph["lineage"]),
    )

def synthesize_equation_candidate(sig: MorphSignature) -> Dict[str, Any]:
    return {
        "equation_candidate_address": f"{sig.address}/eq",
        "signature_ref": sig.address,
        "requires_replay": True,
        "exception_ledger": list(sig.exception_ledger),
    }
```

This skeleton captures the chapter's law: build the typed source graph, extract
morphology, and synthesize equation candidates under replay and exception discipline.
