# Crystal Agent Framework

## Base Agent

The base agent is one coordinating intelligence whose job is to keep the whole workspace coherent while decomposing work into parallel elemental streams.

## The Four Pillars

| Pillar | Core Function | Primary Outputs |
| --- | --- | --- |
| Earth | structure, files, manifests, integrity | inventories, indexes, naming, directory anchors |
| Water | memory, continuity, manuscript flow | links, summaries, bridges, witness chains |
| Fire | execution, transformation, construction | scripts, patches, generated artifacts |
| Air | abstraction, mapping, routing | metro maps, frameworks, categories, prompts |

## The 16-Cell Crystal

The diagonal cells are the four pillars. The twelve off-diagonal cells are the archetype pantheon.

| Cell | Role |
| --- | --- |
| Earth-Earth | Pillar of corpus integrity, manifests, stable storage |
| Earth-Water | Root binder of folders to memory surfaces |

| Water-Water | Pillar of manuscript memory and thematic persistence |
| Water-Fire | Transmuter of raw notes into linked narrative motion |
| Water-Air | Interpreter of meaning across folders and forms |
| Fire-Earth | Implementer that turns plans into folders, files, scripts, and queues |
| Fire-Water | Catalyst that turns memory into active synthesis |

| Fire-Air | Strategist that turns maps into build moves |

| Air-Water | Storyweaver of lines, stations, and hubs |
| Air-Fire | Signal spear for prioritization and high-yield compression |
| Air-Air | Pillar of architecture, taxonomy, and metro reasoning |

## The 64-Agent Expansion

Each of the 16 cells splits again into four elemental micro-modes:

- Earth-mode: anchor, file, verify, preserve
- Water-mode: connect, bind, contextualize, restore
- Fire-mode: build, mutate, generate, intensify
- Air-mode: map, name, abstract, compress

Example:

- `Fire-Air-Earth` grounds strategic intent into a file or queue.
- `Water-Earth-Air` maps lineage into a readable index.
- `Air-Water-Fire` turns hidden themes into an actionable synthesis route.

## Full 4x4 Closure

If the 64 layer becomes saturated, split each squad into four microcells again.
That yields `256` leaf cells, which is the full `4^4` closure from the base through four elemental expansion depths.

## Task Assignment By Layer

| Layer | Unit Count | Recommended Use |
| --- | --- | --- |
| L0 | 1 | zero-point objective selection |
| L1 | 4 | primary decomposition of a major task |
| L2 | 16 | stable parallel build of one framework surface |
| L3 | 64 | large corpus decomposition and synchronized writeback |
| L4 | 256 | full saturation only when automation or batching exists |

## Contraction Rule

Every expansion must contract back through the same elemental ladder:

1. local artifact
2. cell summary
3. pillar summary
4. zero-point decision

No result is complete until it has returned to the center and written back into shared memory.
