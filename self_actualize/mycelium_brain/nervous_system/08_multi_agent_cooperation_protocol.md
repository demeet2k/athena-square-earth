# Multi-Agent Cooperation Protocol

## Purpose

This protocol defines how many agents can build the same nervous system in tandem without fragmenting the corpus.

## Shared Control Surface

All agents should treat the following as common infrastructure:

- `self_actualize\mycelium_brain\toolkit`
- `self_actualize\mycelium_brain\nervous_system`
- `self_actualize\route_quality_ledger.json`
- `self_actualize\corpus_atlas.json`
- `self_actualize\archive_atlas.json`

## Cooperative Division Of Labor

### Earth lane

- atlas refresh
- file manifests
- archive promotion planning
- queue stabilization

### Water lane

- manuscript linkage
- family-level continuity
- cross-manuscript theme extraction
- section-to-chapter synthesis

### Fire lane

- chapter drafting
- code and algorithm artifacts
- executable bridges
- runtime mutation and proof-of-work surfaces

### Air lane

- metro maps
- taxonomy
- routing rules
- prompt and protocol distillation

## Tandem Loop

1. pick one shard
2. check shared queues and rules
3. attempt live Docs if section-draft mode applies
4. read local evidence
5. produce one artifact
6. update the relevant map, queue, receipt, or ledger
7. stop with the next frontier clearly exposed

## Collision Avoidance

If multiple agents touch nearby territory:

- one owns the artifact
- one owns the receipt or ledger
- one owns the map writeback
- one owns the queue update

This prevents silent duplication while preserving parallel throughput.

## Shared Output Contract

Every agent run should leave:

- one artifact
- one local pointer to where it belongs in the crystal
- one statement of what still remains unresolved
