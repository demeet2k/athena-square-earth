# Folder To Basis Embedding

## Embedding law

Every atlas record receives additive bindings only:

- `d3_ingress`: nearest corpus body, line, transfer hubs, and replay reference
- `d4_native`: basis refs and nearest lawful route into the compiled matrix
- `d5_overlay`: compression-state hints only
- `d6_overlay`: overlay-state hints only

These bindings do not replace existing addresses, chapter ids, or appendix ids.

## Deterministic rules

1. Read the record's `top_level`, `relative_path`, heading candidates, excerpt, and role tags.
2. Assign the nearest corpus body station.
3. Score the record against the canonical `16` basis documents.
4. Publish basis refs only when local path/content evidence supports them.
5. Stamp the docs gate and canonicalization status.

`record -> body station -> basis refs -> compiled 4D route -> higher overlay if needed -> same basis-safe return path backward`
