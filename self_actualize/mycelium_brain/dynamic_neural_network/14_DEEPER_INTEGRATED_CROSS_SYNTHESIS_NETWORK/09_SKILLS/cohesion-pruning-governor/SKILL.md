# cohesion-pruning-governor

Use when the request asks to reduce bloat, prune outdated skills, repair lane-stack or `H6` drift, improve router cohesion, or reconcile competing skill generations.

## Workflow

1. Read `../../00_CONTROL/04_ALGORITHMIC_PIPELINE.md`.
2. Read `../../10_LEDGERS/04_PRECEDENCE_AND_ECONOMY.md`.
3. Read `../00_SKILL_ROUTER.md`.
4. Compare the live deep-root skills against:
   - historical mirror skill families
   - precursor deep-net skill surfaces
   - family-local orchestrator skills
   - generated lane, `H6`, and seed bundles
5. Classify each skill as one of:
   - `live authority`
   - `family-local`
   - `historical mirror`
   - `prunable bloat`
6. Repair stale live references, especially any `12_6D_EARTH_CONTROL` path that still masquerades as canonical after `13_6D_EARTH_CONTROL` landed.
7. Update router, manifests, ledgers, and generated bundles so future requests route through one clear precedence order.

## Output Contract

- name the live authoritative skills
- name the historical mirror skills
- name the family-local boundary skills
- name the lane or convergence drift that was repaired
- record what was pruned, deprecated, or rerouted
- update at least one manifest or ledger so the cohesion change survives the session
