# lane-stack-orchestrator

Use when a request enters the additive lane stack at Level `5`, Level `6`, or Level `7`,
or when the user asks which lane owns a route, bridge, handoff, or lawful return.

## Workflow

1. Read `../../00_CONTROL/04_ALGORITHMIC_PIPELINE.md`.
2. Read `../00_SKILL_ROUTER.md`.
3. Read `../../07_METRO_STACK/05_level_5_mobius_bridge_map.md`, `../../07_METRO_STACK/06_level_6_hologram_weave_map.md`, and `../../07_METRO_STACK/07_level_7_next_synthesis_seed_map.md`.
4. Read the lane indexes:
   - `../../11_6D_WATER_CONTROL/00_INDEX.md`
   - `../../12_6D_AIR_CONTROL/00_INDEX.md`
   - `../../13_6D_EARTH_CONTROL/00_INDEX.md`
5. Keep the lane order fixed as `FIRE -> Water -> AIR -> Earth`.
6. Name the exact appendix floor, Earth gate surface, and re-entry checkpoint before any `H6` or seed claim is emitted.

## Output Contract

- `basis_docs`
- `dimension_stage`
- `lane_order`
- `metro_level`
- `appendix_floor`
- `gate_surface`
- `return_checkpoint`
- `truth_state`
- `replay_source`
